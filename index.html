Marshall Labs · Design System v1

A small set of reusable CSS classes.

Drop marshall-labs.css into a page, add class="ml-page" to <body>, and compose with the classes below. Dark by default. Switch modes by setting data-theme="light" on any ancestor.

14 component patterns
6 accent colors
2 themes (dark / light)
0 JavaScript required
1 CSS file to include
01 · Getting started

Drop it in and compose.

Two rules: every page is wrapped in .ml-page and every layout lives inside .ml-container. Everything else is opt-in.

<!doctype html>
<html data-theme="dark">
  <head>
    <link rel="stylesheet" href="marshall-labs.css">
  </head>
  <body class="ml-page">
    <div class="ml-container">
      <h1 class="ml-title">Hello <span class="ml-accent">world</span>.</h1>
    </div>
  </body>
</html>

Start with .ml-card. Add --accent for a left-border, --top for a top-border (stats), --tinted to tint the background with the current accent, --sunken for a slightly inset look.

Sunken card

Accent-left card

Tinted card

07 · Pills & code

Inline tokens and block snippets.

Use .ml-pill for inline code-like tokens (file paths, flags, function names). Color variants add semantic meaning. For multi-line code, use <pre class="ml-code">.

The search flow: Glob "src/**/*.ts" → Grep "handleLogin" → Read auth.ts → Edit. 

const greet = (name) => {
  // say hello to the caller
  return `Hello, ${name}!`;
};

greet("world");
// => "Hello, world!"
<pre class="ml-code">
  <span class="ml-tok-kw">const</span> …
  <span class="ml-tok-fn">greet</span> …
  <span class="ml-tok-str">"world"</span>
  <span class="ml-tok-comment">// …</span>
</pre>

Available syntax spans: .ml-tok-kw, .ml-tok-str, .ml-tok-num, .ml-tok-comment, .ml-tok-fn.

10 · Putting it together

A full section composed from the primitives.

Three accent-left cards with tagged headers — the classic "failure modes" layout.

What goes wrong

Three failure modes as the window fills.

Context rot

Instructions buried in the middle of a long stream lose weight. The model recalls the beginning and end; the middle gets vague.

Seen in the wild

Mid-session you tell the agent "always write a test." Twenty turns later, new code lands without tests.

Context anxiety

As the window approaches full, the model rushes to wrap up. Later steps get shallower; it starts summarizing instead of doing.

Seen in the wild

A 30-file refactor. First 10: careful edits. Last 5: "applied the same pattern to the remaining files."

Cache cascade

Editing early content invalidates every downstream cache block. One small prefix change can 10× your token cost.

Seen in the wild

Someone adds one new MCP tool. The tool list reorders → cached system prefix invalidated → next session bill: ~10×.

None of these are bugs. They're predictable consequences of a flat, position-sensitive, priced-per-token stream.

12 · Theming

Two themes, one attribute.

Set data-theme="dark" (default) or data-theme="light" on <html> — or on any subtree you want to flip. The button in the top-right of this page does it with one line of JavaScript. 

// toggle site-wide
document.documentElement.dataset.theme =
  document.documentElement.dataset.theme === 'dark'
    ? 'light'
    : 'dark';

// or respect OS preference on load
if (window.matchMedia('(prefers-color-scheme: light)').matches) {
  document.documentElement.dataset.theme = 'light';
}

To rebrand, override any of the --ml-* custom properties in your own stylesheet after importing marshall-labs.css. Scope overrides to a theme by nesting under [data-theme="dark"] or [data-theme="light"].
